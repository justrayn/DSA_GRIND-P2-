## Repository Structure

This repository contains my practice and implementation of core **Data Structures and Algorithms (DSA)** concepts. The project is organized to separate fundamental implementations from problem-solving practice.

### Folder Overview

* **DSA GRINDDDD/**

  * **Array List/**
    Contains basic implementations and experiments related to array-based list structures.

  * **Cursor based/**
    Implementations of cursor-based data structures used to simulate linked structures using arrays.

  * **Linked List/**
    Core implementations of singly and/or doubly linked lists along with fundamental operations.

  * **Queue/**
    Queue implementations and related operations.

  * **Stack/**
    Stack implementations and examples demonstrating stack behavior.

  * **problems/**
    A collection of coding problems encountered during practice.
    These problems often combine multiple data structure concepts and algorithmic techniques.

### Additional Files

* **a.c**
  A temporary testing file used for quickly experimenting with code snippets and validating ideas before organizing them into the appropriate folders.

---

### Purpose of This Repository

The goal of this repository is to:

* Practice core **Data Structures and Algorithms**
* Maintain clean implementations of fundamental concepts
* Track solutions to various algorithmic problems encountered during learning

This serves as a personal reference and learning archive for improving problem-solving skills in DSA.
